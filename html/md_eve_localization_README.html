eve_localization

1. Repository Outline

실내배송로봇 framework 1.8, 2.0의 측위 알고리즘을 테스팅, 배포하기 위한 리포지토리.
측위의 정확도와 강성 향상을 위한 여러 실험적인 시도를 하고, validation을 할 수 있는 공간.

2. 모듈 구조

AMCL을 base로 하며 추가적인 노드를 통해 확장성을 지니도록 설계하였음. 노드간 정보 연결 방법은 vanilla nav2_amcl**을 실행하고, 추가적인 패키지가 accuracy, robustness를 위하여 도와주는 구조로 채택함.
아래 항목에서 리포지토리 내의 패키지 단위의 개요를 설명함.

  • **nav2_amcl
    • 직접적으로 로봇 네비게이션 알고리즘에 측위좌표를 넘겨주는 eve_localization 모듈의 뼈대 패키지
    • vanilla amcl은 urg_node + HW interface에서 센서 정보를 받아 2D grid 맵에서 대조하여 위치정보 발행

  • eve_loc_fusion
    • nav2_amcl이 확장성을 지니도록 topic을 조정하는 패키지.
    • 다양한 odometry, localization 알고리즘에 nav2_amcl의 topic, TF domain의 fusion interface를 제공함.
  • eve_kimera_wrapper
    • VIO를 수행하는 kimera-VIO를 ROS2의 센서 인터페이스, topic인터페이스를 사용하여 실행.

3. ROS2 independent c++ 라이브러리

  • openvslam(ROS2 independent library) + openvslam_ros released under GPL license
    1. stereo visual slam으로 global feature map의 생성을 수행
    2. Place Recognition problem을 수행하여 1.의 global feature map에서 global localization을 수행
    3. 의존 라이브러리
      1. Eigen : version 3.3.0 or later
      2. g2o : commit ID 9b41a4e
        1. SuiteSparse
      3. FBoW : Please use the custom version of FBoW released in https://github.com/stella-cv/FBoW.
      4. yaml-cpp : version 0.6.0 or later.
      5. OpenCV : version 3.3.1 or later.
      6. Pangolin : Latest release
        1. GLEW
  • kimera-VIO(ROS2 independent library)
    1. 현재 시간으로부터 queue size에 저장된 이미지, IMU 프레임을 처리하여 visual odometry를 수행하는 라이브러리
    2. eve_kimera_wrapper가 필수적으로 사용하는 코어 라이브러리
    3. 의존 라이브러리
      1. GTSAM >= 4.0
      2. OpenCV >= 3.3.1
      3. OpenGV
      4. GLog, Gflags
      5. Gtest
      6. DBoW2
      7. Kimera-RPGO
      8. ANMS
